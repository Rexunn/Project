CHANGELOG - Racetrack AI Project
=================================

 Week 1 (Nov 30 - Dec 6)
* Getting Started: Kicked off the prototype sprint for the final year project.
* Got the basic Pygame game loop running with arrow-key acceleration and a visual velocity vector.
* Set up a 4-tuple (x, y, vx, vy) for the CarState to make it hashable for A* later.
* Built the first physics engine iteration to check the 9 possible legal acceleration moves.
* Made a quick 9-point scanner to parse PNG track images by pixel color (walls, roads, start/finish)
* Housekeeping: Consolidated repos from "UOLProject" down to just "Project"

 Week 4 (Dec 23)
* Note: Finally back at it after the UoL assessment period took out the last three weeks.
* Track Engine: Rewrote the track parser so it dynamically loads and scales real PNGs at runtime.
* Refactoring: Cleaned up the architecture a lot. Pulled the game engine into game_engine.py to decouple physics from the main loop. 
* Made main.py just handle the View/Controller. Trimmed down CarState to speed up object creation for A*.

 Week 8 (Jan 24)
* A* is Alive: Got the first working A* pathfinder running on the 2D grid!
* Using a priority queue for the open set and Euclidean distance for the heuristic. Wired it into the main loop to visualize the path.
* Bug fix: Spent way too long trying to figure out why the grid wasn't rendering... turned out I flipped the x and y axes when accessing the array (grid[y][x])

 Week 10 (Feb 2-3)
* Added a main menu to make it feel like an actual game.
* Bug fix: A* was silently failing because I was comparing CarState objects directly in the closed set. Swapped to tuple-based hash equality and it works

 Week 11 (Feb 15-16)
* Checkpoints & Physics: Massive updates this week
* Built a flood-fill algorithm to group adjacent yellow tiles into logical checkpoint "islands"
* Upgraded A* to handle lists of coordinates so it can pathfind through multiple stops sequentially
* Bug fix: Fast cars were tunneling through walls between frames. Implemented Bresenham-style raycasting to fix the collision detection

 Week 15 (Mar 12-13)
* Procedural Generation & CPU: Huge week for the architecture. 25] The Genetic Algorithm (GA) is in
* First GA try used 2D integer grids, evolving over 35 generations
* Upgraded the track generation to an 8-waypoint polar ring.Carves corridors between points to make closed circuits.It generates 10/10 solvable tracks in ~8 seconds now
* Implemented full turn-based racing (3-second timer, space to confirm)
* CPUs race at the same time: Easy does random moves, Medium is greedy for checkpoints, Hard uses the precomputed A* path

 Week 16 (Mar 18-19)
* Added an AI stats HUD to show the A* solver's path length, explored nodes, and time
* Made the CPU Hard route visible as a dotted line on the track
* Playtesting showed 3 seconds was too fast, bumped the turn timer to 5s
* Added fun random color themes for generated tracks (tarmac, forest, vice city, snow)

 Week 17 (Mar 23)
* Academic Stuff: Implemented standard BFS just so I have a baseline to compare A* against for the write-up. Added a side-by-side stats screen after each track generation to compare solve times

 Week 18 (Mar 29)
* Visual Polish: Tracks look so much better. Swapped out the angular corridors for Catmull-Rom splines so the tracks have smooth, organic curves. Used a circular brush instead of a square one to fix jagged corners
* Tweaked the GA to reward tracks that force the A* solver to change direction a lot. The tracks are much more demanding now

 Week 19 (Apr 3-6)
* UI & Game Feel: Full UI overhaul—pulsing active checkpoints, live leaderboards, velocity gauges.
* Upgraded checkpoint splitting to 8-way diagonal to fix false island bugs.
* Bug fix: CPUs won't drive backwards anymore. Also, timing out your turn drops your acceleration to zero but keeps your momentum, which feels way more realistic.

 Week 20 (Apr 8-13)
* Multi-lap & Saving: Multi-lap racing works. Upgraded A* to calculate continuous mega-paths across laps without stopping at the finish line. 54]
* Can finally save and load tracks to JSON. Made a menu selector that scans for saved files.
* Encoded the checkpoint sequence directly into the tile values, completely wiping out my geometry sorting bugs.
* Added a quick 'R' restart key to bypass recomputing A*.

 Week 21 (Apr 15-18)
* System Overhaul: Went crazy with 30+ commits.
* Locked down the architecture with a proper GameStateManager FSM so transitions don't break.
* Ripped all the drawing code out of main.py and into a clean ui.py module.
* Added weather! 68] Sunny, Rainy, and Snowy (snow completely limits braking).
* Added hazards: Oil spills randomise acceleration, Potholes cost a life.
* Added wrong-way detection using dot products of your velocity against the track's forward vector. Flashes a banner if you turn around. 73]
* Added a setup screen to tweak GA parameters like waypoint count and track sharpness

 Week 22 (Apr 21-22) 79]
* Ghosts & Leaderboards: Added persistent top-5 leaderboards (MD5 hashing the grids)
* Added Ghost Car replays so you can race your PB
* You can now name your generated tracks before saving them
* Proved my A* heuristic is admissible by upping the divisor

 Week 23 (Apr 23-26)
* Final Polish & Submission
* Swapped heuristic to Chebyshev distance (properly admissible for a 4D state space where diagonals are legal)
* Optimized the solver using heapq
* Player now respawns at the last cleared checkpoint instead of the start line
* Added fading dot-trails behind cars and a marker over the player so you don't lose yourself in the crowd
* Gave the whole UI a dark-panel Flash-style refresh
* Wrote the README.md Project is basically done
